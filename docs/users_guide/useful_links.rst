Useful Links

Community & Governance
----------------------

- `Eclipse SDV Slack <https://sdv.eclipse.org/get-engaged/>`_ — join the community chat to ask questions and connect with contributors
- `Eclipse S-CORE Project Leads <https://projects.eclipse.org/projects/automotive.score/who>`_ — official list of project leads and committers
- `Architecture Community Discussions <https://github.com/orgs/eclipse-score/discussions/categories/architecture-community>`_ — GitHub discussion board for architecture topics
- `Feature Requests & Modifications Board <https://github.com/orgs/eclipse-score/projects/4/views/1>`_ — overview of open feature requests and planned changes

Key Repositories
----------------

- `Eclipse S-CORE GitHub Organization <https://github.com/eclipse-score>`_ — umbrella organization for all S-CORE modules
- `score <https://github.com/eclipse-score/score>`_ — main repository (requirements, architecture, documentation)
- `module_template <https://github.com/eclipse-score/module_template>`_ — template to start a new S-CORE module
- `reference_integration <https://github.com/eclipse-score/reference_integration>`_ — reference QNX integration image and integration tests
- `process_description <https://github.com/eclipse-score/process_description>`_ — S-CORE process description (safety, quality, verification)
- `docs-as-code <https://github.com/eclipse-score/docs-as-code>`_ — Sphinx/sphinx-needs toolchain used across all modules
- `tooling <https://github.com/eclipse-score/tooling>`_ — Bazel rules and macros shared across modules
- `bazel_registry <https://github.com/eclipse-score/bazel_registry>`_ — S-CORE Bazel module registry
